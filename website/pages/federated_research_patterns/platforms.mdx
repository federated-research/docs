# Platform Patterns
There are dozens of federated platforms that could be used by the researcher to perform their federated analytics or federated learning projects. 

Whilst there may be a few different patterns for the types of federation, our expectation is that there wont be many patterns between our the various platforms have decided to implement their approaches. 

## Initiate
A federated analysis or federated learning project usually will start with some command to initiate the project. At this 
stage we are collecting information from platform providers to understand if they are using an API specification that captures how a federated project is initiated.  

Therefore, at this stage, we will list the API specification for the receiving of the analysis to run across the network

## Process
Each TRE will be asked to process some local job, and that local processing can equally be initiated by an API. We have seperated out the API specification that initiates
a federated project and the API specification that is used to initiate a local processing incase these are different.

Therefore, at this stage, we will list the API specification for API that is to request the processing of the analysis with a TRE.
