# Submit a TES task using the TES Wizard


The TES Wizard has three different modes:

<br />
<img src="/images/wizard_submission_mode.png" width="400"></img>

## Simple SQL Query

The Simple SQL Query TES Wizard mode creates a TES task that executes a [Postgres SQL Runner](https://github.com/SwanseaUniversityMedical/5S-TES-analysis-tools/tree/main/SQLRunnerPostgres) container image. To submit a SQL query TES Task you need to set:
- Name
- Description *(optional)*
- TREs to submit to
- A SQL Query

#### Example Submission:
<br />
<img src="/images/wizard_sqlmode.png" width="600"></img>

## Custom Image

The Custom Image TES Wizard guides you through the construction of a Custom TES Task. Details on what a TES Task looks like can be found in the [GA4GH TES API documentation](https://ga4gh.github.io/task-execution-schemas/docs/#tag/TaskService/operation/CreateTask). Enter the following information about the TES Task:
- Name
- Description *(optional)*
- TREs to submit to
- Data Input *(optional)*
- Data Output
- Executors

<img src="/images/wizard_customimage.png" width="600"/>

## Raw JSON

The Raw JSON Tes Wizard mode allows you to submit a TES message as a JSON payload.

#### Example Submission:
<br />
<img src="/images/wizard_json.png" width="600"/>
<Callout type="info" > 

Specify the Project name and TRE you want to submit to in the "tags". To submit to more than one TRE separate the TRE Names using `|`
``` json
"tags": {
    "project": "Testing",
    "tres": "DEMO 01" // or "DEMO TRE 01|DEMO TRE 02"
}
```
</Callout>
